# Instruments

Put reusable instrument sound libraries here so MaixCAM2 demos can use stable paths.

## Layout

```text
instruments/
  drums/
    Shino Drums(sf2)/
      02_Shino_Snare.sf2
      snare_samples/
        ghost.wav
        normal.wav
        accent.wav
  pianos/
    SalamanderGrandPiano-V3/
      SalamanderGrandPiano-V3+20200602.sf2
      samples/
        036.wav
        ...
        095.wav
  guitars/
    Bread Breads Distortion Guitar/
      Bread Breads Distortion Guitar v2.1.sf2
      strums/
        C5_down.wav
        C5_up.wav
    acoustic_guitar/
      FSS-SteelStringGuitar-20200521.sf2
      strums/
        C_down.wav
        C_up.wav
        G_down.wav
        G_up.wav
```

## Notes

- `backend/instrument_server.py` uses these folders by default.
- Snare, piano, electric guitar, and acoustic guitar all play pre-rendered WAV samples first for lower latency and better macOS compatibility.
- Regenerate Salamander piano samples with `python pc_generate_piano_wavs.py` after changing the piano SoundFont.
- Regenerate Shino snare samples with `python pc_generate_snare_wavs.py` after changing the snare SoundFont.
- Electric and acoustic guitar modes play pre-rendered strum WAV files first. Regenerate all guitar strums with `python pc_generate_guitar_strum_wavs.py` after changing either guitar SoundFont.
- Large `.sf2` and `.wav` files are intentionally not generated by the code. Copy or download the files into these folders.
